# SyncGo 跨平台 SSH/SFTP 文件同步工具包设计文档（参考 shuttle 工业级优化版）

## 1\. 项目概述

### 1\.1 项目背景

日常服务器运维、测试部署场景中，需要频繁将本地文件/目录上传至远端 VPS 服务器。传统 scp/rsync 依赖系统命令、跨平台兼容性差、无法嵌入 Go 项目、不支持统一配置管理，且缺少面向 Windows 平台的精细化增量同步策略。

**SyncGo** 是一款基于 Go 语言开发的跨平台配置驱动 SSH/SFTP 文件同步工具，适配 Linux、Windows 全平台，具备快速、稳定的同步能力。本项目参考开源工业级工具 **henryborner/shuttle** 设计思想，纯 Go 原生实现、无系统命令依赖，支持密码/私钥双认证、双同步覆盖策略、增量差分更新，可作为独立 CLI 工具或第三方库嵌入 Go 项目。

### 1\.2 核心目标（新增 shuttle 借鉴能力）

- 跨平台：原生适配 Windows / WSL / Linux / macOS，**Linux、Windows 双向平等兼容，无平台偏向**

- 双认证体系：SSH 密码登录、SSH 私钥登录（支持加密私钥、密钥口令）

- **双模式工业级同步（核心借鉴 shuttle）**：增量覆盖保留远端文件、压缩全量对齐清理远端冗余文件

- 增量同步：基于文件大小/修改时间差分更新，避免全量重传

- 无依赖：纯 Go 实现，不调用系统 scp/rsync，可静态编译单二进制

- 可配置、可复用：结构化配置驱动，支持库调用 \+ 后续封装 TUI/CLI

### 1\.3 适用场景

- Windows 本地开发环境一键同步代码/二进制至远端 Linux VPS

- 自动化部署、测试环境批量文件推送、迭代更新

- Go 项目内嵌轻量化远程文件同步能力

- 隧道、代理工具自动化更新远端程序，区分「保留配置更新」「纯净全覆盖更新」场景

### 2\.1 功能需求

#### 2\.1\.2 核心同步能力（对标 shuttle 核心特性）

借鉴 shuttle 增量同步设计思想，**SyncGo** 针对运维核心痛点，定义两种生产级同步模式，彻底解决「迭代保配置」和「发布强对齐」的核心矛盾：

- **模式一：增量覆盖同步（Overlay Patch）**

    - 仅上传本地新增/修改文件，同名文件覆盖

    - **保留服务端独有文件（配置、密钥、日志）**

    - 适用：业务迭代更新、日常部署、不希望丢失远端自定义配置

- **模式二：压缩全量替换同步（Full Replace）**

    - 本地目录 tar\.gz 打包压缩上传，远端整体解压覆盖

    - **强制对齐目录，删除服务端存在、本地不存在的冗余文件**

    - 适用：纯净发布、二进制整包更新、环境重置、强制本地远端一致场景

#### 2\.1\.3 扩展能力（完全对齐 shuttle 预留特性）

- 文件差分增量同步（mtime/size 比对，跳过无变更文件）

- 自定义文件过滤规则（忽略 \.git、缓存、日志、临时文件）

- 上传进度回调、传输状态可视化

- 配置文件驱动同步（yaml/json 结构化配置，对标 shuttle config\-driven）

- 后续可扩展 TUI 交互界面、双向同步、实时监听同步

## 4\. 整体架构设计

### 4\.1 三层架构（借鉴 shuttle 分层思想）

1. **配置层**：连接配置 \+ 同步策略配置（双覆盖模式、过滤规则、增量开关）

2. **连接层**：SSH/SFTP 连接管理、认证解析、连接复用、异常重连

3. **同步调度层**：根据策略自动路由「增量覆盖 / 全量压缩替换」逻辑，执行文件上传、差分校验、目录对齐

### 4\.2 核心枚举与结构体设计（新增双同步模式）

#### 4\.2\.1 认证类型枚举

- AuthPassword：密码认证

- AuthPrivateKey：私钥认证（文件/原始文本）

#### 4\.2\.2 同步模式枚举（核心新增，对标 shuttle 增量\+全覆盖场景）

- **SyncOverlayPatch**：增量覆盖模式（默认）
        

    - 行为：同名覆盖、新增上传、**保留远端独有文件**

    - 场景：日常迭代、保留远端配置、密钥、日志

- **SyncFullReplace**：压缩全量替换模式
        

    - 行为：本地打包上传、远端解压全覆盖、**清理远端冗余文件**

    - 场景：纯净发布、整包更新、环境重置

#### 4\.2\.3 核心配置结构体

在原有配置基础上新增策略字段，完全配置驱动：

- 基础连接：IP、Port、User

- 认证配置：AuthType、Password、PrivateKeyPath、PrivateKeyRaw、KeyPassphrase

- 同步策略：SyncMode（双模式切换）、Incremental（增量开关）、IgnoreRules（忽略文件规则）

#### 4\.2\.4 同步器 Syncer

持有连接实例与全部配置，统一调度两种同步策略，提供完整文件/目录上传能力与资源管理。

## 5\. 核心模块详细设计

### 5\.1 认证模块

自适应双认证模式，优先解析私钥原始文本，兼容加密私钥，错误链路完整可追溯。

### 5\.2 连接模块

统一建立 SSH \+ SFTP 长连接，支持连接复用，提供安全关闭逻辑，杜绝句柄泄露。

### 5\.3 双模式同步核心实现（项目核心亮点）

#### 5\.3\.1 增量覆盖模式（SyncOverlayPatch）

**核心逻辑**：温和迭代更新，不破坏远端自有文件

1. 递归遍历本地目录，统一转换 Linux 标准路径

2. 远端逐路径检测文件/目录状态

3. 开启增量时：比对 mtime/size，仅上传变更文件

4. 同名文件覆盖、新文件新增、**远端独有文件完整保留**

#### 5\.3\.2 全量压缩替换模式（SyncFullReplace）

**核心逻辑**：强一致性对齐，纯净环境同步

1. 本地目录无损 tar\.gz 打包（纯标准库，跨平台）

2. SFTP 上传临时压缩包至远端

3. 远端执行解压覆盖目标目录，自动清理本地不存在的文件

4. 自动清理本地、远端临时包，无残留垃圾文件

#### 5\.3\.3 双模式差异对比

|同步模式|远端独有文件|同名文件|一致性|适用场景|
|---|---|---|---|---|
|增量覆盖模式|保留|本地覆盖远端|差异更新|业务迭代、保留配置|
|全量替换模式|删除清理|本地覆盖远端|完全一致|纯净发布、环境重置|

### 5\.4 增量差分模块（借鉴 shuttle 核心优化）

通过比对**文件大小、修改时间**，跳过无变更文件，大幅提升重复同步效率，避免无效重传，适配高频迭代场景。
